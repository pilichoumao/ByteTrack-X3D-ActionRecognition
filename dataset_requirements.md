

# 制作动作识别数据集规范

## 1. 数据来源

数据必须 **来自监控摄像头视频**，或使用 **监控摄像头拍摄**。

推荐方式：

* 从真实监控录像中 **裁剪动作片段**
* 或使用监控摄像头在真实场景拍摄

不建议使用：

* 手机近距离拍摄
* 手持移动拍摄

原因：保证训练数据与实际部署环境一致。

---

# 2. 数据样本定义

一个样本为：

**一个视频片段（clip） + 一个动作标签**

每个视频片段 **只能对应一个标签**。

---

# 3. 动作类别

| 类别ID | 类别    | 说明      |
| ---- | ----- | ------- |
| 0    | scoop | 用勺子舀取配料 |
| 1    | pour  | 将液体倒入杯子 |
| 2    | other | 其他所有动作  |

---

# 4. 视频裁剪规则

视频片段必须：

* 包含 **完整动作过程**
* 只对应 **一个主要动作**
* **只有一个人**

---

# 5. 视频时长

推荐：

```
4–8 秒
```

允许范围：

```
3–10 秒
```

动作需要在视频中清晰可见。

---

# 6. 视频技术要求

| 项目  | 要求           |
| --- | ------------ |
| 格式  | mp4          |
| 编码  | H.264        |
| 分辨率 | 720p 或 1080p |
| 帧率  | 30 FPS（必须统一） |
| 方向  | 横向视频         |

禁止：

* 竖屏视频
* FPS不一致
* 严重模糊或损坏视频

---

# 7. 数据量建议

| 类别    | 数量      |
| ----- | ------- |
| scoop | 200–300 |
| pour  | 200–300 |
| other | 300–500 |

建议总量：

```
700–1000 视频
```

---

# 8. 数据目录结构

数据需按以下结构组织：

```
dataset/
│
├── videos/
│   ├── scoop/
│   ├── pour/
│   └── other/
│
├── train.txt
├── val.txt
└── test.txt
```

---

# 9. 文件命名

命名格式：

```
类别_编号.mp4
```

示例：

```
scoop_001.mp4
pour_001.mp4
other_001.mp4
```

---

# 10. 标注文件格式

每行格式：

```
视频路径 类别ID
```

示例（完整示例）

train.txt
```
videos/scoop/scoop_001.mp4 0
videos/scoop/scoop_002.mp4 0
videos/pour/pour_001.mp4 1
videos/other/other_001.mp4 2
videos/other/other_002.mp4 2
```
val.txt
```
videos/scoop/scoop_050.mp4 0
videos/pour/pour_050.mp4 1
videos/other/other_050.mp4 2
```
test.txt
```
videos/scoop/scoop_080.mp4 0
videos/pour/pour_080.mp4 1
videos/other/other_080.mp4 2
```
---

# 11. 数据集划分

推荐比例：

```
train 70%
val   20%
test  10%
```

注意：

同一段监控视频裁剪出的片段 **不能同时出现在不同集合中**。

---

# 12. 数据质量要求

提交数据前需确认：

* 视频可以正常播放
* 动作清晰可见
* 标签正确
* 视频长度符合要求

避免：

* 标签不确定
* 动作不完整
* 视频损坏

---

# 13. 最终交付

提交以下内容：

```
dataset/
videos/
train.txt
val.txt
test.txt
```

压缩为：

```
dataset.zip
```

---


> **优先从监控视频中裁剪数据，而不是重新拍摄。**
